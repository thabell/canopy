# canopy

🇬🇧 [Read in English](README.md)

> **Один тег скрипта. Ноль файлов-ассетов. Бесконечный ландшафт.**  
> Процедурный 3D-ландшафт как живой фон страницы — горы, леса, озёра, смена дня и ночи — всё генерируется на GPU из математики, без растровых текстур.

---

## Боевое применение

`canopy` — фоновая анимация главного экрана **[Крона](https://krona-edu.tilda.ws/)** — онлайн-школы подготовки к ЕГЭ/ОГЭ. Крона — это индивидуальная работа вместо групповых вебинаров, только сертифицированные эксперты ЕГЭ, полный неурезанный материал с дизайнерскими презентациями, карточками, проверкой письменных заданий и аналитикой прогресса. Стек: Django + Wagtail, собственные Figma-плагины, ML-классификация заданий, собственный VPS. Ландшафт работает как Tilda-блок без жалоб на производительность.

## Техническая архитектура

Процедурный 3D-ландшафт реального времени работает на Three.js — движется бесконечно, генерирует горы/воду/леса из шума Перлина прямо на GPU. 10 видов деревьев с уникальной геометрией крон через Worley-noise, инстансинг (2 draw call вместо 20), адаптивное качество под железо пользователя. Дневной цикл, ветер, туман. Встраивается одним скриптом в любой сайт как живой фон. Никаких файлов изображений, никакого пайплайна ассетов, никакой сборки.

### Пять независимых слоёв шума

Высота ландшафта — сумма пяти отдельных FBM-пассов (Fractal Brownian Motion), каждый со своей частотой, амплитудой, количеством октав, лакунарностью и шероховатостью:

| Слой | Роль | Ключевые параметры |
|---|---|---|
| `BASE` | Базовый рельеф / микродетали | f=0.019, A=7, oct=5 |
| `HILL` | Холмы | f=0.013, A=14, oct=6 |
| `MTN` | Горные гребни через ridge-маску | f=0.0062, A=48, oct=8 |
| `LAKE` | Размещение озёрных котловин | f=0.006, порог |
| `FOREST` | Маска покрытия лесом | f=0.008, порог |

`MTN` использует smoothstep ridge-маску перед умножением — острые пики без иглообразных артефактов. `LAKE` продавливает рельеф ниже заданной глубины, когда FBM пересекает порог. Все пять слоёв работают в GLSL vertex shader — CPU не трогает значения высот в процессе рендеринга.

### Ландшафт на GPU

Геометрия — плоский `PlaneGeometry`. Vertex shader вычисляет весь 5-слойный стек на каждую вершину и смещает `y` на месте. Три чанка переиспользуются при движении камеры — никакого выделения новой геометрии в кадр.

### Форма кроны: Worley + Perlin FBM

Каждая крона — икосфера, деформированная в vertex shader смесью **3D Worley noise + 3D Perlin FBM**:

- `crownFreq` — размер ячеек (ниже = крупнее комки)
- `crownNoise` — глубина выдавливания
- `lumpMix` — 0 = чёткие Worley-ячейки, 1 = плавный FBM-рельеф
- `blobSpreadX/Y` — растяжение выходной геометрии (реальное масштабирование)
- `blobYBias` — смещение центра noise-сэмплирования по Y (поникающая / вздёрнутая крона)

Четыре архетипа (`round`, `spreading`, `columnar`, `oval`) задают базовый envelope, поверх которого выдавливаются комки.

### Ветер

Трёхуровневая модель, никакой отдельной логики порывов не нужно.

**Уровень 1 — глобальный вектор ветра (CPU, покадрово):**

```
windVec.x = (sin(t·0.40)·0.60 + sin(t·1.10)·0.25) × strength
windVec.z = (cos(t·0.35)·0.50 + cos(t·0.90)·0.20) × strength
windVec.y =  sin(t·0.30)·0.08 × strength
```

Каждая горизонтальная ось — сумма **двух синусоид с некратными частотами** (0.40:1.10 ≈ 2.75, 0.35:0.90 ≈ 2.57). Когда оба компонента в фазе — амплитуды складываются (порыв), в противофазе — частично гасят (затишье). Нерегулярные порывы возникают сами, без специальной логики. Ось `x` через `sin`, ось `z` через `cos` — сдвиг 90° — направление ветра медленно вращается, а не толкает туда-обратно по одной оси.

**Уровень 2 — пространственная фаза на дерево (vertex shader):**

Раскачивание каждого дерева сдвинуто на `pos_x · 0.10 + pos_z · 0.15`, поэтому порыв проходит по лесу как видимая волна, а не бьёт по всем деревьям одновременно.

**Уровень 3 — локальная нерегулярность кроны (vertex shader):**

```glsl
float sw = sin(lp)·0.70 + sin(lp·2.3 + 0.8)·0.30;
```

Ещё одна пара некратных компонент, уже на уровне каждой вершины — каждая крона получает собственное нерегулярное трепетание независимо от глобального ветра.

### Листовая текстура — без PNG

Fragment shader генерирует паттерны листьев для каждого вида через 2D Voronoi + FBM:
- `round` → ячейки Voronoi + мягкий FBM
- `lobed` → искажённый Voronoi + угловой sin-член
- `pinnate` → растянутое UV-пространство + горизонтальная модуляция полос
- `serrated` → двухслойный Voronoi высокой частоты

Фрагменты ниже порога плотности вызывают `discard` — прозрачные просветы без альфа-текстуры.

### Инстансинг с одним материалом (2 draw call на 10 видов)

Все 10 видов используют **один `ShaderMaterial`**. Параметры вида упакованы в четыре `InstancedBufferAttribute` vec4 на инстанс:

```
aP0: rgb=цвет листвы,  w=тип кроны (0-3)
aP1: blobSpreadX/Y, yBias, crownNoise
aP2: crownFreq, lumpMix, форма листа, плотность листвы
aP3: чёткость краёв, масштаб листа, detailRough, соотношение сторон
```

16 float на дерево. Никакой смены uniform, никакой перепривязки материала. Два патч-меша → **2 draw call для всей геометрии леса**, независимо от видового состава.

### GPU readback высотной карты для расстановки деревьев

Расстановка деревьев на CPU (обход сетки + пороги шума). Проблема: высота в любой точке определена в GLSL, не в JS. Решение: рендеринг fullscreen quad с той же GLSL-функцией высоты в `WebGLRenderTarget`, считывание через `readRenderTargetPixels` в `Float32Array`, билинейная интерполяция для каждой кандидатной позиции. Один GPU bake на рециклинг чанка — никакой копии стека шума на JS.

### Адаптивный пайплайн качества

1. **Probe-фаза** — первые 15 кадров пропускаются (прогрев драйвера), следующие 60 измеряются при минимальных настройках
2. **Выбор пресета** — FPS маппируется на один из 6 пресетов: `ultralow → low → medium → high → ultra → extreme`
3. **Runtime adaptive** — если сглаженный FPS падает ниже `preset.minFPS` дважды за 7-секундные окна, автоматическое снижение на уровень

Переключение пресетов: vertex attributes на патч-геометриях горячо заменяются ссылкой на `IcosahedronGeometry(1,3)` (642 вершины) или `IcosahedronGeometry(0.88,1)` (42 вершины) — никакой перестройки геометрии, никакого перевыделения буферов.

## Система видов

10 листопадных видов с реалистичными диапазонами высот и весами плотности:

`Quercus` · `Tilia` · `Fagus` · `Fraxinus` · `Ulmus` · `Carpinus` · `Alnus` · `Juglans` · `Corylus` · `Castanea`

13 визуальных параметров на вид. Смешанный лес возникает естественно из фильтрации по высоте + density FBM + посевной RNG на ячейку — никаких ручно расставленных объектов. Параметры хранятся в `species-config.js`, редактируются через `crown-tuner.html`.

## Структура репозитория

```
canopy/
├── landscape 2.0.html      ← актуальная версия (Worley кроны, instanced attrs, адаптивное качество)
├── landscape 1.0.html      ← первая версия (сферы-кроны, instanceColor, простой пайплайн)
├── crown-tuner.html        ← интерактивный редактор видов: орбита, слайдеры, экспорт/импорт
├── blender_node_export.py  ← скрипт Blender: экспорт нодовых деревьев в JSON
├── species-config.js       ← актуальные параметры видов (вставить в тюнер или landscape)
├── README.md
└── README.ru.md
```

**`crown-tuner.html`** — standalone Three.js приложение. Все 10 видов рядом, орбитальная камера. Живые слайдеры для каждого crown- и leaf-параметра, экспорт в JSON / копирование как JS. Принимает вставленные конфиги для A/B сравнения.

**`blender_node_export.py`** — запускается в панели scripting Blender. Экспортирует имена нод, типы, значения сокетов и связи из любого материала или GeometryNodes-модификатора в `/tmp/nodes_export.json`. Использовался при прототипировании шума для переноса шейдерных графов из Blender в GLSL. Спасибо за проект и вдохновение [qiri](https://youtu.be/E0tr9SMxvH0)!

**`landscape 1.0.html`** — первая версия. `instanceColor` для цвета видов, простой `SphereGeometry(1,8,5)` без per-instance параметров формы, JS-сторонний сэмплинг высотной карты, фиксированный счётчик деревьев на пресет.

## Использование

Three.js r128 вшит внутри блока скрипта. Достаточно вставить `landscape 2.0.html` (или блок `<div class="landscape">` со скриптом) в любую страницу. Установи `C.DEBUG = true` для живой панели управления (пресеты, цвета биомов, ветер, плотность деревьев, веса видов, редактор зон).
2 патч-меша сделаны для улучшения производительности. Если хотите, чтобы деревья не возникали внезапно, а появлялись из тумана — найдите константу `TREE_CHUNKS` и поменяйте значение 2 → 3.

### Управление цветом биома

Цвет всего ландшафта задаётся через HTML-атрибуты на любых элементах страницы:

```html
<!-- один цвет — применяется один раз при инициализации, цикл смены отключён -->
<span data-landscape-main-color="#2a5c1e"></span>

<!-- несколько цветов — ландшафт плавно переходит между ними в порядке объявления -->
<span data-landscape-main-color="#2a5c1e"></span>
<span data-landscape-main-color="#1a3a6b"></span>
<span data-landscape-main-color="#6b2a1a"></span>

<!-- опционально: количество патчей интерполяции между каждой парой (по умолчанию 8) -->
<span data-landscape-transition-patches="10"></span>
```

Из главного цвета автоматически выводится вся палитра: трава, берег, вода, скалы, горы, снег и цвета каждого вида деревьев. Вода всегда остаётся в синем диапазоне (~210°) независимо от главного цвета. Поведение регулируется `_CTL_WATER_LOCK`

При одном объявленном цвете — никаких накладных расходов на анимацию: палитра вычисляется один раз при загрузке.

## Перспективы

- Смена сезонов: lerp цветов листвы через uniform — геометрия не меняется
- Поверхность воды: плоский меш на уровне `WATER_LEVEL` с wave-шейдером — один draw call
- Хвойные и любые новые виды: Worley-кроны уже умеют любую форму — добавление вида это один объект конфига + ~10 минут в `crown-tuner.html`, никаких изменений в коде, ноль влияния на производительность сколько бы видов ни было (GPU видит один поток instance-атрибутов, а не N видов)
- Звук: пространственный ветер и амбиент можно привязать к существующему uniform ветра
- Адаптивный пайплайн и формат конфига видов готовы к переносу в любой Three.js проект
